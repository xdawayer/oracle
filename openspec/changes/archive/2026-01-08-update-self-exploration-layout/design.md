## Context
- 探索自我页面当前信息密度过高、卡片风格过碎，重点不够突出。
- 蓝紫色占比高导致“AI 感”偏强，需要更低饱和的黑金与疗愈柔和方向。
- 参考图：`/Users/wzb/Pictures/123.png` 与 17:00 批次的 5 张参考图，用于技术附录瀑布式布局。
- Big3 拆分结构参考图：`codex-clipboard-XAFs9B.png`。
- 相位矩阵、行星/小行星图标、宫主星信息需要与参考图一致且可读性更强。

## Goals / Non-Goals
- Goals:
  - 拉开层级，让核心信息更突出、阅读路径更清晰。
  - 降低卡片密度与分割感，形成统一叙事语气。
  - 星盘图成为主流程视觉锚点，位置靠前且更大。
  - 专业附录改为从上到下的瀑布式结构。
- Non-Goals:
  - 不涉及其它页面的布局改造。
  - 不重做移动端体验，仅保证不崩坏。
  - 不引入新框架或全局重构。

## Decisions
- 星盘图位置与尺度
  - 置于“方法论”下方与“核心画像分析”之间。
  - 在现有基础上放大 1.3 倍，作为视觉锚点。
- 核心画像分析结构
  - Big3 拆为 Sun/Moon/Rising 三张独立卡，突出“核心/内在/外在”。
  - 其余四模块（core_melody/top_talent/top_pitfall/trigger_card）统一为 2×2 网格。
- 心理维度模块统一结构
  - 顶部引导语改为 `prompt_question`。
  - 标签替换为更易懂的表达，例如：
    - 模式 → 常见反应
    - 根源 → 关键成因
    - 被触发 → 触发场景
    - 资源 → 缓解方式
    - 练习 → 练习路径
  - 减少分割线和嵌套卡，形成单一阅读流。
- 人生课题与行动（Drive/Fear/Growth）
  - 每个主题呈现“主叙事 + 关键要点 + 行动建议”单一结构，避免碎片化盒子。
- 专业附录瀑布式结构
  - 顺序固定：元素矩阵 → 相位列表 → 行星信息 → 小行星信息 → 宫主星信息。
  - 取消 Tab 或折叠切换，使用明确分段标题与单列布局（默认全展开）。
- 相位矩阵
  - 使用 10 大行星的下三角矩阵，显示相位符号与 orb。
  - 矩阵结构保持三角梯形的视觉秩序，减少列表冗余。
- 行星/小行星图标
  - 使用 SVG 矢量图标，统一风格并与星盘视觉保持一致。
  - 资源集中放置，便于后续替换升级。
- 宫主星信息
  - 使用现代守护星映射，展示守护星所在宫位（飞入宫位）。
- 视觉方向
  - 深色：低饱和黑金为主，减少蓝紫块面。
  - 浅色：疗愈柔和的低对比中性色，保留金色强调。

## Risks / Trade-offs
- Prompt 输出结构变化会影响前后端字段对齐，需要同步更新后端 prompt 与类型。
- 专业附录去 Tab 后页面更长，需要清晰标题与留白防止疲劳。
- 桌面优先可能在窄屏出现密度变化，但不做专门优化。

## Migration Plan
1. 更新后端 prompt schema（natal-overview / natal-dimension / core themes）。
2. 前端调整布局与字段映射。
3. 调整色彩与标题层级样式。
4. 桌面冒烟验证。
