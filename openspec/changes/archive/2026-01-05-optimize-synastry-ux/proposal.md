# Change: 合盘体验全面优化（术语通俗化、排版统一、内容深度增强）

## Why
- 中文状态下仍有大量英文术语，如 "CONJUNCTION/OPPOSITION/SQUARE/TRINE/SEXTILE" 未翻译。
- 对比盘和组合盘的排版过于拥挤，与"探索自我"模块的排版风格不一致。
- 专业占星术语如"敏感面板/修复窗口/修复脚本/微型循环图"对普通用户过于晦涩，降低可理解性。
- AI 生成内容中仍大量使用 "A" 或 "B" 代称，而非用户真实姓名。
- 组合盘解读（关系人格/长期课题/日常相处/综合交叉）过于简单，缺乏深度。
- 对比盘中"敏感面板"等内容应单独拆解解读，而非堆叠展示。

## What Changes

### 1. i18n 翻译补全
- 相位图例翻译：CONJUNCTION → 合相、OPPOSITION → 对冲、SQUARE → 刑相、TRINE → 三合、SEXTILE → 六合。
- 补全所有合盘模块中的英文文案，确保中文场景无英文泄露。
- 与"探索自我"星盘的补充说明保持一致风格。

### 2. 术语通俗化
将以下专业术语替换为用户更易理解的表述：
- "敏感面板" → "情感敏感区" 或 "内心触发点"
- "修复窗口" → "最佳沟通时机"
- "修复脚本" → "沟通话术" 或 "修复建议"
- "微型循环图" → "互动模式" 或 "相处循环"
- "滋养点" → "相处加分项"
- "触发点" → "容易踩雷的地方"
- 其他晦涩术语一并梳理替换。

### 3. AI 内容姓名替换
- 修改 synastry prompts，要求 AI 使用用户真实姓名而非 A/B。
- 在 prompt 中明确传入 `nameA` 和 `nameB`，并在 system prompt 中强调使用姓名。
- 前端兜底：若 AI 输出仍包含 A/B，进行字符串替换。

### 4. 排版格式统一
参考"探索自我"模块的排版规范：
- 统一字号层级：标题 → 正文 → 辅助说明。
- 统一间距：模块之间、卡片之间、内容行之间。
- 统一颜色：标签色、正文色、辅助色。
- 移除过于紧凑的网格布局，改为更有呼吸感的垂直排列。
- 对比盘中"敏感面板"由 5 列紧凑网格改为逐项展开的卡片布局。

### 5. 内容深度增强
- **对比盘**：敏感面板中每个维度（月亮/金星/火星/水星/深层）单独成卡，包含更详细的解读。
- **组合盘**：
  - 关系人格部分增加 outer/inner/growth 三维度的详细解读。
  - 长期课题部分增加 nodes（北交/南交）和 chiron（凯龙）的解读。
  - 日常相处部分 maintenance_list 增加到 5-8 条具体建议。
- 修改对应的 AI prompts，要求生成更详细、更有层次的内容。

### 6. 间距调整
- 修复可能存在的隐藏元素（如背景框）阻挡间距调整的问题。
- 星盘与下方内容之间的间距对齐"探索自我"模块。
- 各 Section 之间的间距统一为 `space-y-8` 或 `mt-8`。

## Impact
- Affected specs:
  - `specs/generate-synastry-report/spec.md`
- Affected code:
  - `constants.ts` — i18n 翻译补全与术语替换
  - `App.tsx` — 排版调整、组件间距、卡片布局
  - `backend/src/prompts/manager.ts` — AI prompts 优化（姓名传入、内容深度）
  - `backend/src/api/synastry.ts` — 姓名传递到 AI context
  - `components/AstroChart.tsx` — 相位图例翻译
- Dependencies:
  - 与 `optimize-synastry-report` 变更互补，不冲突。
  - 需在 `optimize-synastry-report` 的 A/B 替换任务完成后进行 prompt 优化。
- Notes:
  - 术语通俗化需保持专业性与可理解性的平衡，避免过度简化。
  - 英文模式下保留原有术语，仅中文模式做通俗化处理。
