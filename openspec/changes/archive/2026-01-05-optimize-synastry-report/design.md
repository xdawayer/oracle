# 合盘报告优化设计说明

## 设计目标
- 统一合盘与 Ask Oracle 的加载体验，减少等待割裂感。
- 让每个 tab 的内容结构更清晰：顶部图表（全息除外）、正文、底部方法论。
- 文案统一为“人名驱动”，中文场景避免无意义英文与 A/B 代称。
- 字体与“探索自我”一致，提升可读性。

## 布局与信息流
- 顶部区：tab 标题（保留 tab 按钮标签）。
- 内容区：图表（全息总览不显示） → 主要正文模块。
- 底部区：方法论/免责声明统一放在每个 tab 末尾。
- 顶部 AI 来源提示条移除（`reportMeta?.source === 'ai'` 绿色提示框）。

### 综述卡片子标题移除
移除每个 tab 综述区域的"标题 — 副标题"行，仅保留正文内容。

示例（移除前）：
```
全息总览 — 关系的深度体检
不仅仅是简单的吉凶判断。这是一份关于你们能量互动的深度核磁共振...
```

示例（移除后）：
```
不仅仅是简单的吉凶判断。这是一份关于你们能量互动的深度核磁共振...
```

各 tab 同理处理。

## 加载动效
- 合盘分析加载态复用 Ask Oracle 动画（同视觉语言与节奏）。
- 抽取为共享 Loading 组件，支持不同场景的文案轮播配置。
- 合盘专属 loading_phrases（10+ 条），示例：
  - "两盘交织中..."
  - "缘分轨迹显现..."
  - "星轨正在对接..."
  - "能量场同步中..."
  - "关系密码解读中..."
  - "行星对话开启..."
  - "命盘共振分析..."
  - "互动模式识别..."
  - "深度连接扫描..."
  - "双人能量解码..."
  - "轨道交汇点定位..."
  - "关系图谱生成中..."

## 姓名替换与文案策略
- UI 文案中所有 “A/B” 改为真实姓名：
  - tab 标题、描述、正文提示、占星附录标签。
- AI 文案中通过 prompt 强制使用姓名，避免 A/B 代称。
- 若姓名缺失：回退显示 A/B；历史内容不做替换。

## 星盘展示方案
**复用现有 `AstroChart` 组件**，使用 `optimize-chart-settings` 变更中已定义的星盘生成规则与配置：

- 本命盘 tab：
  - `<AstroChart type="natal" profile={profileA|B} />`
  - 使用 `NATAL_CONFIG` 配置。
- 对比盘 tab：
  - `<AstroChart type="synastry" profile={profileA} partnerProfile={profileB} />`（或反向）
  - 使用 `SYNASTRY_CONFIG` 配置（bi-wheel 双人盘）。
- 组合盘 tab：
  - `<AstroChart type="composite" profile={profileA} partnerProfile={profileB} />`
  - 使用 `COMPOSITE_CONFIG` 配置（中点盘）。
- 数据来源：复用 synastry 技术数据（natal_a / natal_b / composite / syn_ab / syn_ba）。
- 视觉规范：遵循"探索自我"的尺寸、配色与相位线样式。

## 字体与 i18n
- 同步“探索自我”字体规范：
  - 标题：Serif；正文：标准文本；避免斜体。
  - 不使用过小字号（避免 text-xs 作为正文）。
- 中文场景尽量使用中文术语，必要时可保留标准占星名词英文。

## 相位表展示
- 对比盘相位表一次性全量展示，取消横向/纵向滚动容器。
- 小屏可折行，但必须完整呈现全部条目。

## 数据来源约束
- **禁止 mock 数据**：合盘报告的所有动态内容（星盘、AI 文案、技术数据）必须来自真实计算或 AI 生成，不使用 mock 数据。
- 静态 UI 文案（标题、标签、提示语）除外。
- 若 API 或 AI 调用失败，显示明确的错误提示，而非回退到 mock 内容。

## 风险与回退
- 若双人盘或组合盘数据缺失：显示错误提示，不使用 mock 占位。
- 若 AI 文案仍出现 A/B：仅在新生成场景提示改用姓名，不对已缓存的 AI 响应做替换。
- 若星盘渲染失败：显示加载错误状态，不显示空白或 mock 图表。
